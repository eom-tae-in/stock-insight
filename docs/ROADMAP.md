# StockInsight ROADMAP

이 문서는 현재 코드 기준으로 앞으로 정리해야 할 작업만 남긴 실행 로드맵이다.

## 우선순위 원칙

1. 문서와 실제 코드의 불일치 제거
2. 사용자 경험이 깨지는 문제 우선
3. 구조를 크게 뒤엎기보다, 실제 병목과 불편을 줄이는 방향 우선
4. 배포 기준 안정성을 유지한 상태에서 개선

## Phase 1. 문서 정리

목표: 기준 문서를 현재 구현과 맞춘다.

- `README.md` 를 소개/설치/실행/배포 개요 중심으로 유지
- `PRD`, `TRD`, `ROADMAP`, `API_CONVENTIONS`, `CLAUDE.md` 만 핵심 문서로 관리
- 오래된 guides 문서와 보조 SQL 문서는 제거 또는 정리

수락 기준:

- 살아 있는 문서만 남는다
- README와 기준 문서가 서로 충돌하지 않는다

현재 상태:

- 핵심 6개 문서 체계로 정리 진행 중
- 남은 작업은 세부 문구 보정과 커밋 반영

## Phase 2. 키워드 분석 UX 안정화

목표: 키워드 조회 경험을 더 분명하게 만든다.

- 조회 중, 조회 실패, rate limit UI를 유지/개선
- 실패 시 재시도 흐름을 더 명확하게 다듬기
- 저장 성공/실패, 최신화 성공/실패 토스트 품질 점검
- 키워드 검색/결과/상세 화면의 버튼 배치와 상태 일관성 유지

수락 기준:

- 키워드 조회 흐름에서 사용자가 현재 상태를 헷갈리지 않는다
- 실패 시 빈 화면처럼 보이지 않는다

## Phase 3. 성능 체감 개선

목표: 첫 조회 체감을 줄인다.

- 키워드 결과 페이지의 부가 API 호출 필요성 재검토
- Redis hit/miss 기준 로그와 체감 지연 구간 정리
- Trends 조회 경로에서 실제 느린 구간을 계측할 수 있도록 준비
- 필요 시 캐시 TTL, prewarm, 경량 DTO 검토

수락 기준:

- “느리다”는 문제를 추상적으로 보지 않고 구간별로 설명할 수 있다
- 큰 구조 변경 없이도 개선 가능한 항목이 분리된다

## Phase 4. API/서비스 구조 정리

목표: 현재 실제 구조를 유지하되, 책임 분리를 더 분명히 한다.

- API route에서 서비스/DB/외부 호출 책임을 다시 점검
- 현재 남아 있는 직접 조합 로직을 서비스 레이어로 더 밀어낼지 검토
- 인증/응답 형식 공통 처리 일관성 재점검

수락 기준:

- 주요 API들이 비슷한 패턴으로 읽힌다
- 새 기능 추가 시 기존 패턴을 재사용할 수 있다

## Phase 5. 키워드/종목 상세 기능 정리

목표: 저장된 분석 데이터 관리 기능을 안정화한다.

- 키워드 상세 최신화/오버레이 흐름 점검
- 종목 상세/미리보기/저장 흐름 점검
- 다운로드 품질과 파일명 규칙 정리

수락 기준:

- 저장된 데이터 복원, 최신화, 삭제, 다운로드가 회귀 없이 동작한다

## Phase 6. 테스트 기반 강화

목표: 핵심 플로우를 자동 검증할 수 있게 만든다.

- 순수 계산 로직 단위 테스트
- 인증/입력 검증/API 응답 통합 테스트
- 로그인, 종목 조회, 키워드 조회, 저장, 복원 정도의 E2E 스모크 테스트

수락 기준:

- 핵심 회귀를 자동으로 잡을 수 있다

## 현재 우선순위 요약

1. 문서 정리 마무리
2. 키워드 UX 마감
3. 성능 체감 분석과 소폭 개선
4. 구조 정리
5. 테스트 도입
