# StockInsight MVP PRD (로컬 저장소 기반)

## 핵심 정보

**목적**: 특정 종목의 5년 가격 흐름과 대중 검색 관심도를 비교하여 투자 판단에 활용. 조회한 종목 데이터를 로컬에 저장하여 재사용하고 관리할 수 있다.

**사용자**: 혼자 사용하는 로컬 개발 환경에서, 특정 종목의 5년 데이터와 대중 관심도를 반복해서 분석하는 개인 투자자.

**운영 환경**: `npm run dev`로 실행되는 로컬 Next.js 개발 환경 (배포 없음)

## 사용자 여정

```
1. 대시보드 (/)
   - 저장된 종목 카드 그리드 확인
   - 최초 방문 시: "저장된 종목이 없습니다. [+ 추가]를 눌러 첫 종목을 추가하세요." 안내
   ↓ [카드 클릭]                                    ↓ [+ 추가 버튼 클릭]

2-A. 종목 상세 (/analysis/[id])                  2-B. 새 종목 조회 (/search)
   - DB에서 저장된 데이터 로드                      - 서비스 소개 및 사용 예시 문구
   - 지표 요약 카드 5개 표시                       - Ticker 심볼 입력 (예: AAPL, TSLA, MSFT)
   - 차트 3종 (주가+MA13, Trends, 비교) 검토       - 빈 값 제출 방지, 영문 대문자 자동 변환
   - 엑셀/PNG 다운로드                            ↓ [조회 버튼 클릭 또는 Enter]

                                               3. 데이터 수집 및 저장 (로딩 상태)
                                                  - Skeleton UI 표시 (전체 섹션)
                                                  - 진행 메시지:
                                                    주가 수집 중 (F003)
                                                    → Trends 수집 중 (F004)
                                                    → 지표 계산 중 (F005, F006)
                                                    → DB 저장 중 (F013)
                                                  ↓ [수집 결과 분기]

                                                  [성공] → 종목 상세 페이지 (/analysis/[id])로 리다이렉트
                                                           수집한 데이터를 즉시 차트와 지표로 표시
                                                           사용자가 결과 확인 후 대시보드로 복귀 (뒤로가기)

                                                  [실패] → 에러 메시지 표시
                                                           "잘못된 ticker", "API 응답 실패" 등
                                                           [다시 시도] 버튼 (현재 페이지 유지)

4. 대시보드 카드 호버
   ├── [새로고침] 버튼 (F016)
   │   - 해당 종목 최신 데이터 재수집 시작
   │   - 카드 위치 유지, 카드 내 로딩 스피너
   │   - 완료 시 카드 데이터 최신값으로 갱신
   │
   └── [삭제] 버튼 (F015)
       - 삭제 확인 다이얼로그 표시
       - [확인] → DB에서 cascade 삭제
       - 카드 그리드에서 즉시 제거

5. 브라우저 뒤로가기 (종목 상세 → 대시보드)
   - 대시보드로 복귀
```

## 기능 명세

### 1. MVP 핵심 기능

| -------- | --------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ---------------------------------------------------- | ----------------------------- |
| **F001** | Ticker 입력 및 조회         | 종목 심볼을 입력받아 데이터 수집을 트리거. 빈 값 방지, 영문 대문자 자동 변환 포함                                                                                                                                                                                                                                                                                                                                                                                        | 서비스의 모든 기능이 이 입력에서 시작됨              | 새 종목 조회 페이지, 대시보드 |
| **F002** | 오늘 종가 조회              | 해당 ticker의 최신 종가를 단일 수치로 표시                                                                                                                                                                                                                                                                                                                                                                                                                               | 현재 시점의 기준 가격 제공                           | 종목 상세 페이지              |
| **F003** | 5년 주간 주가 조회          | Yahoo Finance API로 최근 5년간 주간 종가 수집 (`interval: 1wk`). Yahoo Finance 주간 기준일(금요일)을 date-fns의 `startOfISOWeek()`로 ISO 주 시작일(월요일)로 정규화하여 저장. **개발 단계에서 실제 API 응답으로 오프셋 검증 필요**                                                                                                                                                                                                                                       | 장기 흐름 분석의 기초 데이터                         | 종목 상세 페이지              |
| **F004** | 5년 주간 Google Trends 조회 | SerpAPI Google Trends로 최근 5년간 주간 검색 관심도(0~100) 수집. 검색 키워드는 yahoo-finance2의 `quoteSummary()`로 조회한 companyName 우선 사용, 조회 실패 시 `"{ticker} stock"` 폴백. 수집된 날짜(일요일 기준)를 `startOfISOWeek()`로 월요일로 정규화. **SerpAPI timestamp의 정확한 기준(주 시작/종료)에 따라 1주 오프셋 가능 → 개발 단계에서 실제 API 응답 1건 확인하여 정규화 로직 검증 필수. F009 비교 차트에서는 ISO week number 기반 매칭으로 1주 이내 오차 허용** | 서비스의 차별화 핵심 데이터                          | 종목 상세 페이지              |
| **F005** | MA13 계산                   | 주간 주가를 기반으로 13주 이동평균선 계산                                                                                                                                                                                                                                                                                                                                                                                                                                | 중기 추세 파악의 핵심 지표                           | 종목 상세 페이지              |
| **F006** | YoY 계산                    | 52주(1년) 기준 전년 동기 대비 주가 변화율 계산. 상승/하락 색상 구분                                                                                                                                                                                                                                                                                                                                                                                                      | 연간 성과 비교를 위한 핵심 지표                      | 종목 상세 페이지              |
| **F007** | 주가 + MA13 차트            | 5년 주간 종가(파란색)와 MA13(주황색)을 겹쳐 표시하는 라인 차트                                                                                                                                                                                                                                                                                                                                                                                                           | 가격 추세와 이동평균 시각화                          | 종목 상세 페이지              |
| **F008** | Google Trends 차트          | 5년 주간 검색 관심도(0~100)를 시계열 영역 차트로 표시                                                                                                                                                                                                                                                                                                                                                                                                                    | 대중 관심도 변화 시각화                              | 종목 상세 페이지              |
| **F009** | 주가 vs 트렌드 비교 차트    | 주가와 검색 관심도를 이중 Y축으로 겹쳐 표시하여 상관관계 시각화. 두 데이터는 ISO week number(YYYY-Www) 기준으로 동일 주 데이터를 매칭하여 시간축 정렬                                                                                                                                                                                                                                                                                                                    | 두 데이터의 상관관계 직관적 확인                     | 종목 상세 페이지              |
| **F010** | 지표 요약 카드              | 현재 종가, MA13, YoY, 52주 최고가, 52주 최저가를 카드 형태로 요약 표시                                                                                                                                                                                                                                                                                                                                                                                                   | 수치 기반 빠른 판단 지원                             | 종목 상세 페이지              |
| **F011** | 엑셀 다운로드               | 원본 주가 데이터, Google Trends 데이터, 계산 지표를 .xlsx 파일로 저장                                                                                                                                                                                                                                                                                                                                                                                                    | 추가 분석을 위한 데이터 내보내기                     | 종목 상세 페이지              |
| **F012** | 차트 PNG 다운로드           | 표시된 차트 3종을 개별 또는 일괄 PNG 이미지로 저장                                                                                                                                                                                                                                                                                                                                                                                                                       | 보고서/공유용 차트 추출                              | 종목 상세 페이지              |
| **F013** | 조회 결과 로컬 DB 저장      | 수집/계산된 데이터 전체를 로컬 SQLite DB(`data/stock-insight.db`)에 저장. `searches` 테이블에 종목 메타데이터 및 핵심 지표 저장, `price_data` 테이블에 5년 주간 주가 배열 저장, `trends_data` 테이블에 5년 주간 트렌드 배열 저장. 동일 ticker 재조회 시 `searches` 레코드를 UPSERT하고 `price_data`/`trends_data`는 전체 교체(DELETE + INSERT). 저장 완료 시 생성된 search_id를 반환                                                                                     | 데이터 재사용 및 대시보드 관리를 위한 핵심 저장 기능 | 새 종목 조회 페이지           |
| **F014** | 저장된 종목 목록 조회       | DB `searches` 테이블에서 전체 저장 종목 목록을 조회하여 대시보드 카드 그리드로 표시. 각 카드에는 ticker, company_name, current_price, yoy_change와 스파크라인용 최근 52주(1년) `price_data` 배열을 포함. 정렬 기준: `searched_at` 최신순. 종목 없을 때는 안내 메시지 표시                                                                                                                                                                                                | 로컬 저장 데이터의 진입점이자 서비스의 홈 화면       | 대시보드                      |
| **F015** | 종목 삭제                   | 대시보드 카드 호버 시 삭제 버튼 표시. 클릭 시 확인 다이얼로그 표시 후 사용자 확인 시 DB에서 해당 search_id의 `searches`, `price_data`, `trends_data` 레코드 전부 삭제(CASCADE). 삭제 완료 후 카드 그리드에서 즉시 제거                                                                                                                                                                                                                                                   | 불필요한 종목 제거 및 DB 용량 관리                   | 대시보드                      |
| **F016** | 저장 데이터 새로고침        | 대시보드 카드 호버 시 새로고침 버튼 표시. 클릭 시 해당 ticker에 대해 F003(주가 재수집) → F004(트렌드 재수집) → F005~F006(지표 재계산) → F013(DB 갱신) 순서로 처리. 처리 중 카드 위치 유지, 카드 내 로딩 스피너 표시. 완료 시 카드 데이터(current_price, yoy_change, 스파크라인 등)를 최신값으로 갱신하고 `last_updated_at` 업데이트                                                                                                                                      | 5년 데이터 로컬 캐시를 최신 상태로 유지              | 대시보드                      |

### price_data 테이블 (5년 주간 주가 시계열)

| 필드      | 설명                                                                                                            | 타입                              |
| --------- | --------------------------------------------------------------------------------------------------------------- | --------------------------------- |
| id        | 레코드 고유 식별자                                                                                              | INTEGER PRIMARY KEY AUTOINCREMENT |
| search_id | searches.id 외래 키. CASCADE DELETE 적용                                                                        | INTEGER NOT NULL                  |
| date      | ISO 주 시작일(월요일) 기준 "YYYY-MM-DD" 형식. Yahoo Finance 원본(금요일)을 date-fns `startOfISOWeek()`로 정규화 | TEXT NOT NULL                     |
| close     | 해당 주 종가 (USD)                                                                                              | REAL NOT NULL                     |

**인덱스**: `(search_id, date)` 복합 인덱스 (상세 페이지 차트 데이터 조회 성능)

### trends_data 테이블 (5년 주간 Google Trends 시계열)

| 필드      | 설명                                                                                                      | 타입                              |
| --------- | --------------------------------------------------------------------------------------------------------- | --------------------------------- |
| id        | 레코드 고유 식별자                                                                                        | INTEGER PRIMARY KEY AUTOINCREMENT |
| search_id | searches.id 외래 키. CASCADE DELETE 적용                                                                  | INTEGER NOT NULL                  |
| date      | ISO 주 시작일(월요일) 기준 "YYYY-MM-DD" 형식. SerpAPI 원본(일요일)을 date-fns `startOfISOWeek()`로 정규화 | TEXT NOT NULL                     |
| value     | 해당 주 검색 관심도 (0~100)                                                                               | INTEGER NOT NULL                  |

**인덱스**: `(search_id, date)` 복합 인덱스

### 테이블 관계

```
searches (1) ──── (N) price_data
       (1) ──── (N) trends_data
```

searches 레코드 삭제 시 연결된 price_data, trends_data 자동 CASCADE DELETE.

### 클라이언트 상태 (기존 유지, 역할 변경)

클라이언트 상태는 DB 저장 이전의 임시 수집 결과 보관에만 사용. 저장 완료 후 DB가 단일 진실의 원천(SSOT). 기존 StockData, TrendsData, CalculatedMetrics 타입은 API 응답 및 DB 저장 전 중간 DTO로 계속 사용.

## 기술 스택 (최신 버전)

### 프론트엔드 프레임워크

- **Next.js 15.5** (App Router + Turbopack)
- **React 19** (최신 동시성 기능)
- **TypeScript 5.6+**

### 스타일링 & UI

- **TailwindCSS v4** (설정파일 없는 새로운 엔진)
- **shadcn/ui** (Card, Skeleton, Button, Badge)
- **Lucide React** (아이콘)

### 차트

- **Recharts 3.x** (React 19 공식 호환)
  - `LineChart` - 주가 + MA13 차트
  - `AreaChart` - Google Trends 차트
  - `ComposedChart` - 주가 vs 트렌드 비교 차트 (이중 Y축)
- **html-to-image** - DOM 기반 차트 PNG 변환

### 폼 & 검증

- **React Hook Form 7.x** - Ticker 입력 폼 상태 관리
- **Zod** - Ticker 심볼 유효성 검사 스키마 (1~10자 영문자)

### 데이터 수집 (서버사이드)

- **yahoo-finance2** - Yahoo Finance API 클라이언트 (주가 데이터, `interval: 1wk`)
- **SerpAPI Google Trends API** - 검색 관심도 수집 (공식 API, 5년 주간 데이터 자동 제공). `q` 파라미터에 companyName 우선 사용, 실패 시 "{ticker} stock" 폴백
- **Next.js Route Handlers** - 서버사이드 API 엔드포인트 (`/api/stock`, `/api/trends`, `/api/searches` 등)
- **date-fns** - 날짜 정규화 (`startOfISOWeek()` 사용, Yahoo Finance 금요일 / SerpAPI 일요일 → ISO 월요일로 통일)

### 데이터 저장 (서버사이드)

- **better-sqlite3** - 로컬 SQLite DB 클라이언트. npm 패키지로 설치, 별도 DB 서버/앱 설치 불필요. 동기 API로 Next.js Route Handlers에서 직접 사용. 저장 위치: 프로젝트 루트 `data/stock-insight.db`. 5년치 260개 주간 데이터 \* 다수 종목 저장에 충분한 용량 제공. `next.config.ts`에 `serverExternalPackages: ['better-sqlite3']` 설정 필요. **DB 초기화 시 반드시 `PRAGMA foreign_keys = ON` 실행 (외래 키 CASCADE DELETE 정상 작동 필수)**

### 다운로드

- **xlsx (SheetJS)** - 엑셀 파일 생성 및 브라우저 다운로드
- **html-to-image** - 차트 DOM 노드를 PNG Blob으로 변환

### 개발 환경

- **Node.js 20+**
- **npm 10+**
- **localhost:3000** (`npm run dev` 실행)
- **data/** 디렉터리 - SQLite DB 파일 저장 위치 (`.gitignore`에 `data/*.db` 추가)

### 환경 변수 설정

프로젝트 루트에 `.env.local` 파일 생성:

```
SERPAPI_KEY=your_serp_api_key
```

- **SERPAPI_KEY**: SerpAPI Google Trends API 호출을 위한 인증 키. [https://serpapi.com](https://serpapi.com)에서 회원가입 후 API 키 발급 필요. 로컬 개발 환경에서만 사용하므로 공개 저장소에 업로드하지 말 것 (`.gitignore`에 `.env.local` 추가됨)
- **.env.example**: 팀원/배포 시 참고용으로 빈 키 값 제공 (예: `SERPAPI_KEY=` )
