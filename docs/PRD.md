# StockInsight PRD

## 1. 제품 정의

StockInsight는 개인 사용자가 종목 가격 흐름과 Google Trends 검색 관심도를 함께 보고, 저장된 종목과 저장된 키워드를 반복 분석할 수 있도록 만든 웹 애플리케이션이다.

현재 제품은 두 개의 축으로 구성된다.

- 종목 분석
- 키워드 분석

인증은 Supabase Auth를 사용하고, 사용자 데이터는 Supabase PostgreSQL에 저장한다.

## 2. 제품 목표

- 사용자가 종목과 키워드를 사용자별로 안전하게 저장하고 다시 볼 수 있어야 한다.
- 종목 가격 데이터와 Google Trends 데이터를 같은 주간 축으로 비교할 수 있어야 한다.
- 저장된 키워드 분석에 종목 오버레이를 추가해 흐름을 비교할 수 있어야 한다.
- 조회 중, 실패, rate limit 상태를 사용자에게 명확하게 보여줘야 한다.
- 다운로드, 삭제, 최신화 같은 반복 작업이 안정적으로 동작해야 한다.

## 3. 비목표

- 실시간 시세 스트리밍
- 투자 추천 또는 자동 매매
- 뉴스/커뮤니티 감성 분석
- 다중 사용자 협업
- 모바일 네이티브 앱

## 4. 핵심 사용자

- 종목과 키워드의 장기 흐름을 비교하려는 개인 사용자
- 반복 조회와 저장 기능이 중요한 사용자
- 로그인 후 자기 데이터만 관리하는 단일 계정 사용자

## 5. 핵심 사용자 여정

### 5.1 인증

1. 사용자는 회원가입 또는 로그인한다.
2. 보호된 페이지에 접근하면 로그인 여부를 검사한다.
3. 로그인 후 원래 가려던 페이지로 복귀할 수 있어야 한다.

### 5.2 종목 분석

1. 사용자는 `/search` 에서 티커 또는 회사명을 검색한다.
2. 앱은 종목 미리보기를 생성한다.
3. 사용자는 미리보기에서 저장하거나 상세 페이지로 이동한다.
4. 저장된 종목은 `/stock-analysis` 에서 다시 조회할 수 있다.
5. 종목 상세에서는 지표, 차트, 표를 본다.
6. 필요하면 최신화 또는 삭제한다.

### 5.3 키워드 분석

1. 사용자는 `/keyword-analysis/new` 에서 키워드를 입력한다.
2. 앱은 `/keyword-analysis/search` 로 이동한 뒤 실제 Trends 조회를 시작한다.
3. 조회 중에는 로딩 UI가 보여야 한다.
4. 성공 시 트렌드 차트와 지표를 보여준다.
5. 실패 시 빈 상태가 아니라 명확한 실패 UI를 보여준다.
6. 사용자는 키워드를 저장하고, 나중에 저장된 키워드를 복원할 수 있어야 한다.

### 5.4 키워드 오버레이 및 상세 분석

1. 사용자는 저장된 키워드 분석에 종목 오버레이를 추가한다.
2. 오버레이는 키워드 분석 단위로 저장된다.
3. 사용자는 키워드 상세 페이지에서 상단 차트, 오버레이, 커스텀 목록을 본다.
4. 오버레이를 삭제하거나 최신화할 수 있어야 한다.

### 5.5 다운로드

1. 사용자는 종목 분석 또는 키워드 분석 결과를 Excel로 내보낼 수 있다.
2. 차트를 PNG로 저장할 수 있다.
3. 다운로드 데이터는 화면에 보이는 분석 기준과 일치해야 한다.

## 6. 기능 요구사항

### F001. 인증 및 사용자 격리

- Supabase Auth를 사용한다.
- 사용자 데이터는 `auth.uid()` 기준으로 분리한다.
- 보호된 페이지와 보호된 API는 인증 실패 시 각각 적절한 동작을 해야 한다.

### F002. 종목 저장 및 조회

- 사용자별 저장 종목 목록을 제공한다.
- 동일 사용자 기준 중복 종목 저장은 중복 생성 대신 재사용 또는 갱신한다.
- 종목 상세 분석 페이지를 제공한다.

### F003. 종목 시계열 수집

- Yahoo Finance 기반 5년 주간 데이터를 수집한다.
- 데이터는 저장 후 재사용 가능해야 한다.
- 최신화 기능을 제공한다.

### F004. 키워드 Trends 조회

- Google Trends 기반 5년 데이터를 조회한다.
- 조회 중 로딩 상태를 보여준다.
- rate limit과 일반 실패를 구분해 보여준다.
- Trends 조회 실패를 빈 성공 응답으로 숨기지 않는다.

### F005. 키워드 저장 및 복원

- 사용자는 조회 성공한 키워드를 저장할 수 있다.
- 저장된 키워드 목록을 다시 볼 수 있다.
- 저장된 키워드를 다시 열 때 기존 분석을 복원할 수 있다.

### F006. 키워드 상세 분석

- 저장된 키워드별 상세 분석 페이지를 제공한다.
- 분석 조건과 오버레이 목록을 볼 수 있어야 한다.
- 키워드 분석 최신화 기능을 제공한다.

### F007. 오버레이 관리

- 키워드 분석에 종목 오버레이를 추가할 수 있다.
- 오버레이는 분석 단위로 저장된다.
- 삭제, 정렬, 최신화가 가능해야 한다.

### F008. 다운로드

- 종목/키워드 분석 데이터를 Excel로 내보낼 수 있어야 한다.
- 차트를 PNG로 저장할 수 있어야 한다.

## 7. 데이터 모델 요구사항

핵심 저장 모델은 다음과 같다.

- `searches`
- `stock_price_data`
- `keywords`
- `keyword_analysis`
- `keyword_stock_overlays`
- `overlay_chart_timeseries`

legacy keyword 테이블은 확장하지 않고 제거 방향을 유지한다.

## 8. 품질 요구사항

- `npm run typecheck`
- `npm run lint`
- `npm run build`

위 기본 품질 게이트를 통과해야 한다.

프로덕션 UI는 내부 에러 메시지를 그대로 노출하지 않아야 한다.

## 9. 성공 기준

- 사용자가 종목 분석과 키워드 분석을 로그인 후 안정적으로 사용할 수 있다.
- 키워드 조회 성공/실패/제한 상태가 UI에서 분명히 드러난다.
- 사용자별 저장 데이터 격리가 유지된다.
- 배포 환경에서 `/api/pytrends` 기반 Trends 수집이 동작한다.
