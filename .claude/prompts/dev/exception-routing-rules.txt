[Exception Routing Rules]

현재 문제 상황을 아래 기준으로 판단하고, 올바른 단계로 돌아가세요:

1. 요구사항이 이상하거나 맞지 않음
→ PRD 확인

2. 기능 정의 / API / 데이터 구조가 이상함
→ TRD 확인

3. 현재 코드 구조와 작업이 맞지 않음
→ starter 이후 프로젝트 구조 기준으로 판단

4. Task 범위가 애매하거나 너무 큼
→ ROADMAP에서 Task 재확인 또는 재분해

5. plan 결과가 이상함
→ Task 크기 문제 → task-sizing-rules 적용

6. execute가 범위를 넘거나 이상하게 구현됨
→ 현재 Task 범위 다시 확인 후 plan부터 재시작

7. 여러 방향으로 갈 수 있어 판단이 어려움
→ ROADMAP의 구현 순서 기준으로 결정

절대 하지 말 것:
- 임의로 구조 변경 ❌
- 새로운 기능 추가 ❌
- Task 범위 확장 ❌

항상 현재 Task 범위 안에서 해결하세요.
