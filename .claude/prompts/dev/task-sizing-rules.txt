[Task Sizing Rules]

현재 Task가 적절한 크기인지 아래 기준으로 판단하세요:

1. 이 Task는 하나의 명확한 목적만 가지는가?
2. 한 번의 작업 흐름으로 구현 가능한가?
3. 여러 레이어(DB + API + UI)를 한 번에 크게 건드리지 않는가?
4. Task 이름이 추상적이지 않은가? (예: "로그인 기능 구현" ❌)
5. 완료 기준이 명확한가? (구현 완료 여부를 판단할 수 있는가?)
6. 이 Task를 그대로 plan/execute에 넣어도 문제가 없는가?

다음 중 하나라도 해당하면 Task를 더 쪼개세요:

- 여러 기능이 섞여 있음
- 작업 범위가 너무 넓음
- 구현 순서가 불명확함
- 완료 기준이 애매함

올바른 예:
- users 테이블 생성
- 회원가입 API 엔드포인트 생성
- 비밀번호 해싱 로직 추가

잘못된 예:
- 로그인 기능 구현
- 사용자 시스템 만들기

현재 Task가 적절하지 않다면, 더 작은 Task로 분해하세요.
