[Project Execution Rules]

이 프로젝트는 PRD → TRD → starter → ROADMAP 기준으로 진행됩니다.

현재 작업은 하나의 Task 단위로 수행됩니다.
다른 Task나 Phase의 작업을 절대 포함하지 마세요.

다음 규칙을 반드시 지키세요:

1. 현재 프로젝트 구조를 먼저 분석한 후 작업하세요
2. starter에서 결정된 구조와 기술 스택을 변경하지 마세요
3. TRD에 없는 기능을 추가하지 마세요
4. ROADMAP에 정의된 Task 범위를 벗어나지 마세요
5. 여러 작업을 한 번에 수행하지 말고, 현재 Task 하나만 집중하세요
6. 불필요한 리팩토링이나 구조 변경을 하지 마세요
7. 구현 후에는 해당 Task의 완료 기준을 만족해야 합니다

작업 중 판단이 애매하면:
- 구조 문제 → 현재 코드 구조 기준
- 기능 범위 → ROADMAP 기준
- 요구사항 → TRD 기준

지금부터 현재 Task 하나에 대해서만 plan을 작성하세요.
